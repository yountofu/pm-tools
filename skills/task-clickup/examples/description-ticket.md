# 티켓 Description 작성 예시

> [UI] 계정 관리 : 비밀번호 변경 태스크에 작성된 실제 예시

---

## 기능 요약

현재 비밀번호 확인 → 새 비밀번호 + 확인 입력으로 비밀번호를 변경한다. 현재 비밀번호를 알고 있는 본인만 변경 가능.

## 주요 스펙

### 접근 권한

| 권한 조건 | 설명 |
|-----------|------|
| 로그인 사용자 | 계정 관리 → 로그인 및 보안 → "비밀번호 변경" |

### 입력 검증

| 항목 | 필수 | 규칙 |
|------|------|------|
| 현재 비밀번호 | Y | 필수 입력 |
| 새 비밀번호 | Y | 영문+숫자+특수문자 포함, 8~16자 |
| 새 비밀번호 확인 | Y | 새 비밀번호와 일치해야 함 |

### 비즈니스 규칙

- BR-AM03: 현재 비밀번호, 새 비밀번호, 새 비밀번호 확인을 모두 입력해야 한다
- BR-AM04: 새 비밀번호는 영문, 숫자, 특수문자 포함 8~16자여야 한다
- BR-AM16: 새 비밀번호와 새 비밀번호 확인이 일치해야 한다

## 오류 처리

- 새 비밀번호 요구사항 미충족 → 체크리스트로 미충족 항목 표시 (인라인)
- 새 비밀번호 확인 불일치 → "비밀번호가 일치하지 않아요." (인라인)
- 현재 비밀번호 오류 → "비밀번호 변경에 실패했어요." (인라인 에러 박스)
